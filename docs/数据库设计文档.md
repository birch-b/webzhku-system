# 淘宝购物系统 - 数据库设计文档

## 一、设计概述

### 1.1 设计目标
为淘宝购物系统设计统一的数据库架构，支持四类用户角色（浏览者、顾客、商家、运营商）的完整业务流程，确保数据一致性和扩展性。

### 1.2 设计原则
- 遵循第三范式（3NF），减少数据冗余
- 使用外键约束保证数据完整性
- 合理创建索引优化查询性能
- 统一命名规范，便于团队协作

### 1.3 角色权限说明
| 角色 | 标识 | 权限范围 | 对应模块 |
|------|------|----------|----------|
| 浏览者 | browser | 仅浏览公开页面（首页、商品列表、公告） | 前台浏览 |
| 顾客 | customer | 购物车、订单、支付、评价、地址管理 | 组员3 |
| 商家 | shopkeeper | 店铺管理、商品管理、订单处理、评价回复 | 组员2 |
| 运营商 | operator | 用户管理、店铺审核、数据统计、公告发布 | 组员1 |

## 六、数据库操作规范

### 6.1 表名规范
- 使用小写英文字母和下划线
- 采用复数形式表示（如 `users`、`orders`）

### 6.2 字段名规范
- 使用小写英文字母和下划线
- 禁止使用MySQL保留字
- 外键字段统一命名为 `关联表名_id`（如 `user_id`、`shop_id`）

### 6.3 数据类型规范
| 用途 | 推荐类型 | 说明 |
|------|----------|------|
| ID字段 | BIGINT | 支持大数据量 |
| 用户名/密码 | VARCHAR(50)/VARCHAR(128) | MD5密码固定32位 |
| 金额 | DECIMAL(12,2) | 精确计算 |
| 时间 | DATETIME | 包含日期和时间 |
| 状态 | TINYINT | 0-255足够存储状态值 |
| 长文本 | TEXT | 富文本内容 |

### 6.4 建表脚本管理
